01Strategy

Strategy summary

Dieses Issue baut die Komponenten-Schicht des Grüne-AT Design Systems auf eine bereits vollständige, stabile Token-Basis (31 --gat--Properties aus Issue #3). Die Richtung: ein flacher, single-class gat--Komponenten-Layer unter dem bestehenden :root — ausschließlich token-getrieben, reines CSS, kein JavaScript.

  • Token-Basis ist komplett — mit einer Lücke.

    Farben, Schriften, Typo-Skala, Zeilenabstände und Abstände existieren und sind WCAG-kontrastgeprüft. Es fehlen Radius-, Border- und Container-Tokens — ein kleiner, klar kommentierter Ergänzungsblock schließt sie.

  • Das gelieferte Logo ist monochrom dunkelgrün.

    SVG mit einer einzigen Füllfarbe #2d793c — auf einem dunkelgrünen Header unsichtbar. Lösung: Header dunkelgrün, Logo per CSS auf Weiß invertiert. Keine zweite Datei nötig.

  • CD-Regel „Text auf Grün“ strukturell erzwingen.

    Karten- und Header-Komponenten paaren immer eine grüne Fläche mit dem passenden on-*-Token.

Primary recommendation: CSS Grid fürs Seiten-/Container-Raster, Flexbox für komponenten-interne Achsen; Responsivität rein über minmax()/clamp() plus wenige max-width-Breakpoints; Störer als reines transform: rotate()-Badge; Header dunkelgrün mit CSS-invertiertem Logo.

02Flow

Task flow & dependencies

Empfohlener Aufbau-Pfad innerhalb von design-system.css — vom Fundament zu den Komponenten:

03Decisions

Key decisions & options

Raster-System: Grid vs. Flexbox

Empfohlen

CSS Grid für das zweidimensionale Seiten-/Container-Raster (repeat(auto-fit, minmax()) bricht ohne Media-Query um), Flexbox für eindimensionale Achsen (Header-Zeile, Navigation, Button-Gruppen).

Nur Flexbox — eindimensional; echtes Spalten-Raster mit gleichen Zeilenhöhen braucht Grid.

Helle Logo-Variante

Offen — User

Das gelieferte SVG ist monochrom (#2d793c). Empfehlung: per CSS auf Weiß invertieren (mask + background-color, oder filter: brightness(0) invert(1)) — keine zweite Datei nötig.

User um eine native helle SVG-Datei bitten — wäre Qualitätsverbesserung, aber nicht erforderlich; blockiert unnötig.

Header-Hintergrundfarbe

Offen — User

Dunkelgrün empfohlen: CD-konform („Text auf Grün“), Logo invertiert auf Weiß, Text in on-primary.

Weißer Header — würde Header-Text auf Weiß setzen und damit die CD-Grundregel verletzen.

Ergänzende Tokens (Radius/Border/Container)

Don't hand-roll

Issue #3 hat nur das CD-Vokabular getokenisiert; Radius, Border-Width und Container-Breite fehlen. Statt Werte hartzucodieren: kleiner, klar als „Web-Ergänzung, Issue #4“ kommentierter --gat--Token-Block am Ende des :root — bewahrt das Acceptance-Kriterium „keine hartcodierten Größen“.

Störer-Akzentfarbe

Offen — User

Gelb (highlight) oder Magenta (accent) — beide CD-konform. Der Plan kann beide als Modifier anbieten (.gat-stoerer--gelb / --magenta).
