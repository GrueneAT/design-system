01Strategy

Read this first — the rest is detail.

Strategy summary

Die heutige Skelett-index.html wird zu einem vollständigen, deutschsprachigen Style Guide ausgebaut — eine einzige statische HTML-Seite ohne JavaScript. Leitlinie: die Seite dogfooded design-system.css — Kopf, Navigation/TOC, Raster und jedes Komponentenbeispiel werden mit den echten gat--Klassen gerendert, nicht abgemalt.

  • Sechs Abschnitte = sechs Tasks.

    Die Seitengliederung (Farben, Typografie, Layout, Komponenten, CD-Layout, Spezifikation) bildet 1:1 die Task-Sequenz ab — deklarativer, sequentieller Aufbau.

  • Lückenlose Abdeckung ist das Kernkriterium.

    Alle 22 --gat-*-Tokens und alle 22 .gat-*-Klassen aus design-system.css müssen im Guide erscheinen — ein Verifikationsschritt grept das CSS dagegen.

  • Echte Komponente vs. doc--Chrome.

    Swatches, Code-Blöcke und Do/Don't-Raster sind Doku-Möbel — doc--präfixiert im inline <style>, nie als gat--Klasse getarnt.

  • Kein ausführbares JavaScript.

    Der einzige <script> ist der application/json-Datenblock; jede literale </script-Sequenz im JSON wird gebrochen.

02Flow

Task flow & dependencies

Task 1 Gerüst, doc-Style, Anchor-TOC Inline doc-Style (D-2), gat-header als TOC, sechs leere Sections.
Task 2 Abschnitt Farben 14 Farb-Tokens als Swatches im gat-grid.
Task 3 Abschnitt Typografie 11 Typo-Tokens + 4 gat-Typo-Klassen live.
Task 4 Abschnitt Layout 5 Layout-Klassen, Abstände, Maße, Breakpoints.
Task 5 Abschnitt Komponenten Header/Nav, Buttons, Karten, CD-Elemente live + Markup.
Task 6 CD-Layout, JSON-Spec, Footer D-1 JSON-Spec-Block, CD-Prinzipien, Schlussprüfung.

Tasks laufen sequentiell: jeder füllt einen Abschnitt der in Task 1 angelegten index.html. Task 6 ist die lasttragende Stufe — sie liefert die maschinenlesbare Spezifikation und die Schlussprüfung auf wohlgeformtes HTML.

03Decisions

Key decisions & options

D-1 — Maschinenlesbare Spezifikation

Chosen

Eingebetteter <script type="application/json">-Block in index.html — eine Datei, kein Sync-Risiko.

Separates design-system.json — zweite Datei, die mit dem CSS synchron gehalten werden müsste.

D-2 — Doc-spezifisches CSS

Chosen

Doc-Chrome (Swatches, Code-Blöcke, Do/Don't-Raster) als inline <style> mit doc--Präfix — hält das gehostete Stylesheet sauber und die Grenze zu echten gat--Komponenten sichtbar.

Doc-Chrome in design-system.css — verwässert das öffentliche Komponenten-Vokabular.

Navigation ohne JavaScript

Chosen

Native <a href="#anchor">-TOC plus CSS scroll-behavior: smooth und scroll-margin-top — voll funktionsfähig ohne JS.

JS-gestützte Navigation / Smooth-Scroll — von CONTEXT verboten.
