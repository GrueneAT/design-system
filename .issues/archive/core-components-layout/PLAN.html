Strategy summary

Issue #3 delivered the --gat- token vocabulary; this plan adds the component layer on top, purely additively — a flat, single-class CSS layer appended below the existing :root. Nothing in the token base changes.

The one real design tension is the logo: the delivered SVG is monochrome dark green (#2d793c) and would vanish on the dark-green header (D-5). Resolution — copy the SVG into the repo under a URL-safe name and recolor it to white via CSS mask + background-color, so no second asset file is needed.

  • Extend, don't restructure.

    Components are appended below a complete, stable token :root — flat single-class selectors, no naked tag selectors that would bleed into consuming tools.

  • Close the token gap, don't hardcode.

    Issue #3 has no radius/border/container token. Task 1 adds a small, clearly-commented supplementary --gat- block so the "no hardcoded sizes" criterion holds.

  • CD rule enforced structurally.

    "Text only on green" is baked into .gat-header and .gat-card--primary/secondary — a green surface and its matching on-* token are always paired.

  • No JavaScript anywhere.

    Responsiveness via repeat(auto-fit, minmax()), flex-wrap, clamp() and a minimal set of max-width media queries.

02Flow

Task flow & dependencies

Task 1 Foundation: @import, logo asset, supplementary tokens Font @import placed above :root; logo copied URL-safe; radius/border/container tokens.
Task 2 Layout system + Header/Navigation Grid/container, dark-green header, CSS-recolored white logo, wrapping nav.
Task 3 Typography classes Headline/Subline/Fliesstext/Emphasis with CD line-height tokens.
Task 4 Button components Primary fill + secondary outline, token radius and border-width.
Task 5 Card / surface components Green-surface variants pairing surface + on-* token (CD rule).
Task 6 CD elements + final verification Underline, highlight, Stoerer (gelb + magenta); whole-file checks.

Tasks run sequentially. Task 1 is the foundation — tokens and the logo asset must exist before any component references them. Task 2 is the load-bearing task (header + layout, the most CD-sensitive surface). Tasks 3–6 each append an independent component group and assume Tasks 1–(N−1) are complete.

03Decisions

Key decisions & options

Logo recolor on the dark-green header

Chosen

CSS mask + background-color: var(--gat-color-weiss) on the monochrome SVG — token-driven, vector-sharp, no second asset, user not blocked (D-2/D-4).

filter: brightness(0) invert(1) on an <img> — works but the white cannot come from a token.

Ask the user for a native light logo SVG — an avoidable blocker; a later optional quality improvement.

Layout engine

Chosen

CSS Grid for the 2D page/container raster, Flexbox for 1D component-internal axes (header row, nav, button groups).

Flexbox-only — cannot do an equal-height column raster. Grid-only — cannot do clean linear wrap.

The radius/border/container token gap

Chosen

Add a small, clearly-commented supplementary --gat- token block at the end of :root (radius / border-width / container-max / breakpoint).

Hardcode 4px etc. directly in components — violates the "no hardcoded sizes" acceptance criterion.

Stoerer accent color

Both delivered

Ship both modifier classes — .gat-stoerer--gelb and .gat-stoerer--magenta — per locked decision D-6; the consuming tool picks per context.
