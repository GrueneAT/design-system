01Strategy

Strategy summary

Reine Dokumentations- und Beispiel-Arbeit auf einem inhaltlich fertigen Repo. Die kritische Forschungsfrage — wo überall Verweise auf das persönliche System liegen — hat ein eindeutiges Ergebnis: genau eine Produktdatei ist betroffen, README.md, Zeilen 22–33. index.html und design-system.css sind bereits sauber.

  • Vier klar trennbare Pakete.

    README umarbeiten, CHANGELOG anlegen, Minimal-Beispiel anlegen, Style-Guide-Abschnitt ergänzen — jedes ein eigener Commit.

  • Etablierte Standards statt Eigenbau.

    CHANGELOG.md folgt „Keep a Changelog“ 1.1.0 + SemVer 2.0.0; das Minimal-Beispiel bindet die absolute gehostete CSS-URL ein (nicht relativ).

  • Gefahr: zu breiter Such-/Ersetzungsschritt.

    Ein grep/sed ohne --exclude-dir=.issues beschädigt 12 Artefaktdateien, deren flomotlik-<link> korrekt ist. Jede Verifikation schließt .issues/ aus.

02Flow

Task flow & dependencies

Task 1 README.md umarbeiten personal-system-Block löschen, Stand ersetzen, Einbindung verfeinern.
Task 2 CHANGELOG.md anlegen Keep a Changelog, Eintrag [1.0.0] - 2026-05-22.
Task 3 examples/minimal.html anlegen Vollständiges Minimal-Beispiel, absolute Hosted-CSS-URL.
Task 4 index.html ergänzen Integrations-Abschnitt vor dem Footer + Nav-Link.

Reihenfolge

Tasks laufen sequenziell. Task 3 ist die lasttragende Arbeit (das dogfoodende Beispiel); Task 4 verlinkt es. Task 1 und 2 etablieren die Doku-Verweise, auf die Task 4 zeigt.

03Decisions

Key decisions & options

Bezug zum persönlichen System (D-1)

Chosen

Bestehende Verweise repo-weit aus den Produktdateien entfernen — betroffen ist nur README.md (Zeilen 22–33). .issues/ bleibt unangetastet.

Querverlinkung persönliches ↔ org-System (ursprünglicher ISSUE.md-Punkt) — ein öffentliches Org-Repo soll nicht auf das private System des Maintainers verweisen.

Versionierung (D-2)

Chosen

Feste Latest-URL + CHANGELOG.md im „Keep a Changelog“-Format mit Semantic Versioning. Start bei 1.0.0 (D-4).

Versionierte URLs / Pfade (/v1/design-system.css) — per D-2 ausgeschlossen.

Git-Tag fuer v1.0.0 (D-5)

Note

Es wird kein Git-Tag gesetzt. Das CHANGELOG.md verzichtet daher bewusst auf Compare-/Release-Links — diese wären ohne Tag 404.

Minimal-Beispiel als echte Datei (D-3)

Chosen

examples/minimal.html als echte, über GitHub Pages erreichbare Datei, die das System real dogfooded — absolute gehostete CSS-URL, kein Build.

Relative ../design-system.css-Einbindung — würde das Hosting nicht dogfooden und Konsumenten ein falsches Muster zeigen.
